# SPEC: source_id 方式（引用と出典の機械的1対1対応）

本書は、教材の引用・転記数値を出典表の各行と機械的に結ぶ識別子 `source_id` の仕様を定める。
背景は docs/METHODOLOGY.md・docs/OPS.md・docs/RELEASE_RECORD_20260717.md「既知の制限」2 が「未導入の改善課題」として挙げていた項目で、本SPECはその**基盤の敷設**にあたる。

## 1. 目的

各教材の「転記数値の出典一覧」表の各行に一意なIDを与え、将来、教材本文の引用箇所から `[[SRC-...]]` で当該出典行を機械的に参照できるようにする。これにより、誤りが見つかったときに「その数値がどの資料から来たか」を人手の照合なしで追える。

## 2. ID書式

```
SRC-<unit>-<file>-NN
```

- `<unit>`: 単元ディレクトリ名（例 `jhs-soc-civics-market-price`）。教科・学年・単元を含みリポジトリ内で一意。
- `<file>`: 出典表を含むファイルの識別トークン。`teacher_notes.md` は `main`、`teacher_notes_<suffix>.md` は `<suffix>` を小文字化し `[a-z0-9-]` 以外（全角の「以降」等）を除去したもの（例 `teacher_notes_L04-06.md` → `l04-06`、`teacher_notes_L04以降.md` → `l04`）。
- `NN`: そのファイル内での2桁ゼロ詰め連番（`01` から）。**ファイルごとに 01 から採番**する。

`<unit>`+`<file>` を含めることで、同じ単元に複数の出典表ファイルがあってもID全体はリポジトリ内で一意になる（既存の unit_id 命名とは接頭辞 `SRC-` で明確に区別される）。

**書式妥当性**（CIが検査）: 正規表現 `^SRC-[a-z0-9]+(?:-[a-z0-9]+)*-[0-9]{2,}$` に一致すること。すなわち小文字英数字のセグメントをハイフンで連結し、末尾が2桁以上の数字であること。

例: `SRC-jhs-math-3-square-roots-main-01`、`SRC-jhs-math-3-similar-figures-s3s4-02`

## 3. 出典表の列

各出典表の先頭に `source_id` 列を置く。

```
| source_id | 資料名 | 年度 | 指標名・用途 | URL | 確認日 | 頁・箇所 |
```

既存6列（資料名・年度・指標名・用途・URL・確認日・頁・箇所）の内容は不変。付与は列の追加とID採番のみで、資料名・数値・URL・確認日・頁は1文字も変更しない。

## 4. 本文からの参照（back-reference）

教材本文の引用箇所からは、将来的に二重角括弧記法 `[[SRC-...]]` で出典行を指す。例（本文中）: 「通過率は約4割だった `[[SRC-jhs-math-3-square-roots-main-01]]`」。

**この本文側の紐付けは段階導入とする**（正直な明記）:

- **新規単元**: 制作時点から本文にID参照を付ける。
- **既存単元**: 大量のインライン引用を一括で書き換えることはしない。教育本文の破壊リスクが最大の作業のため、各単元の改稿・レビュー時に順次付与する。

したがって本SPEC導入直後の本文参照数は 0 が正常であり、参照が無いこと自体は不整合ではない。残量は curriculum/SOURCE_ID_COVERAGE.md で可視化する。

## 5. CI 検査（tools/ci_checks.py 8番目）

1. **書式妥当性**: すべての `source_id` が §2 の正規表現に一致すること。違反で fail。
2. **ファイル内一意性**: 同一ファイルの出典表内でIDが重複しないこと。違反で fail。
3. **本文参照の整合（実在検査のみ）**: `materials/` 配下の本文中に `[[SRC-...]]` 参照があり、その参照先IDがどの出典表にも実在しない場合のみ fail。実在IDへの参照は OK、参照が1件も無いことも OK（段階導入のため）。

CIは全 `source_id` の集合（全出典表から収集）を実在判定の台帳として用いる。現状は本文参照 0 件のため pass する。

## 6. 対象範囲外（このSPECでやらないこと）

- 教材本文（lesson_*.md）の既存インライン引用を `[[SRC-...]]` 参照へ一括置換すること（§4 のとおり段階導入）。
- 出典表の内容（資料名・数値・URL・確認日・頁）の変更。

# 仕様: 図版の生成方式

本書は、本リポジトリの全図版に適用する生成方式・描画規約・検証規約を定めるものです。

## 1. 生成方式 = Pythonスクリプトによるパラメトリック生成（コード来歴）

- 図版は**手描き・画像編集・プロンプト生成ではなく、Pythonスクリプトが座標計算してSVGを出力**する。図版はすべて生成スクリプト由来のSVGであり、借用画像は使わない。
- 理由: プロンプト来歴（「こう頼んで作った」）より**コード来歴**（「このコードを実行すれば同じ図が再現できる」）の方が、第三者による改修・再生成が確実にできる。
- 生成スクリプトは各単元の `assets_provenance/generate_figures.py`。
  - **外部ライブラリ不使用**（Python標準ライブラリのみ）。`python3 generate_figures.py` だけで全図を再生成できる。
  - 相似比・平行・中点・角度などの座標は**コード内で厳密に計算**する（目分量の座標直書きをしない）。
  - スクリプト内に **assert（自己検証）** を置き、生成のたびに「図が数学的に正しいか」「本文の数値と一致しているか」を自動チェックする。assertが1つでも失敗すれば図は出力されない。
- 出力先は各単元の `assets/`（SVGのみ）。

## 2. 生成来歴の格納規約

```text
（単元フォルダ）
├── assets/                        # 図版本体（SVG）。ビュー生成器はここだけを見る
│   └── L06_fig1_parallel_segment_ratio.svg
└── assets_provenance/             # 生成来歴。学習者向け成果物には含めない
    ├── generate_figures.py        # 生成スクリプト＝一次来歴。改修→再実行で全図再生成
    └── FIGURE_MANIFEST.md         # 図版台帳（対象レッスン・意図・パラメータ・生成日・検証結果）
```

- 各SVGの**先頭コメント**にも「生成スクリプト名・図ID・生成日」を埋め込む（ファイル単体が迷子になっても来歴へ戻れる）。
- 図版台帳は**スクリプトが自動生成**する（手書きしない）。台帳と実ファイルのずれを構造的に防ぐため。
- 図の修正は必ず「スクリプトを直す→再実行」で行う。**SVGの直接編集は禁止**（来歴が切れるため）。

## 3. ファイル命名

```text
L{レッスン番号2桁}_fig{連番}_{slug}.svg
例: L01_fig1_similar_triangles.svg
```

レッスン番号はゼロ埋め2桁。連番はレッスン内の登場順。slugは英小文字＋アンダースコアで内容を表す。

## 4. 描画規約（白黒印刷対応が最優先）

**大原則: 色に意味を持たせない。** 白黒印刷（グレースケール）で情報が失われないこと。

| 項目 | 規約 |
|---|---|
| 線色 | 黒 `#000` のみ。塗りの区別は**濃淡グレー＋ハッチング（斜線パターン）** で行う |
| 背景 | 白ベタ `<rect fill="#fff">` を敷く（ダークモード画面でも紙と同じ見えを保証＝画面・紙等価） |
| 主線（図形の辺） | 実線 幅1.6。強調辺は幅3.2 |
| 補助線 | 破線 `stroke-dasharray="6 4"` 幅1.1 |
| 寸法線 | 細実線＋両端ティック、ラベルは線の中央 |
| 対応する辺 | **ティック（小さな交差線）の本数**で対応を示す（1本/2本/3本） |
| 等しい角 | **弧の本数**で示す（1重/2重/3重） |
| 平行 | **矢羽（>）の個数**で示す（1個/2個） |
| 直角 | 小さな正方形マーク |
| 頂点・分点 | 黒丸ドット r≈2.5。頂点名ラベルは図形の外側に置く |
| 装飾 | 過剰装飾なし。影・グラデーション・立体風表現は禁止。実物（木・池など）は最小限の輪郭線のみ |

## 5. ラベル・文字・答え非記載

- **フォント指定なし**（`font-family` を書かない＝閲覧環境のシステム既定に委ねる）。日本語ラベル可。
- 文字サイズはviewBox幅の**約3%以上**（A4横幅いっぱいの印刷で実寸約5mm以上になり判読できる）。
- 本文と同じ記号・同じ数値・同じ向きを使う。**本文と矛盾する図は不可**。
- 与件が多い図は、図の下のキャプション行にまとめてよい（図の中は点名＋マークだけにして視認性を守る）。
- **求める量は「？」で示し、答えの数値は図に書かない**（解答分離の原則と整合）。加えて、解答の数値が図に漏れていないかを禁止文字列の機械検索で検査する（答え漏れの機械検査）。

## 6. SVG技術要件

- ルート要素に `xmlns` と **`viewBox` を必ず指定**。`width`/`height` は指定しない（埋め込み側で相対制御＝レスポンシブ）。
- **自己完結**: 外部参照（外部画像・外部CSS・Webフォント・外部`<use>`）は一切なし。ハッチングは `<defs><pattern>` で内蔵。
- 円弧は座標サンプリングした `<path>` 折れ線で描く（arcフラグの向き事故を防ぎ、コードの検証可能性を上げる）。
- 1図=1ファイル。複数概念の並置は1ファイル内のパネル分割で行う。
- **`<title>`/`<desc>` にAI再利用用のメタ情報を埋め込む**: 図の意図・主要数値・同型の図をAIに描かせるための説明文をSVG内に持たせ、図単体でも機械が文脈を読めるようにする。

## 7. 図の内容規約

- 図は**該当レッスン本文の記述・数値・記号と完全一致**させ、一致をassertで検算する（本文の数値をスクリプトのパラメータにそのまま書き、計算結果が本文の答えと一致することを生成時に確認する方式）。
- 縮尺が意味を持つ図（測量など）は実寸比どおりに描く。概念図は比率が正しければ絶対寸法は自由。
- 対応の順序（∽の並びなど）を図でも守る。

## 8. 領域別の追補規約

全領域共通: コード来歴・白黒規約・答え非記載・`<title>`/`<desc>` のメタ情報を標準装備したうえで、次を追加する。

1. **関数グラフ系**: 通過点・目盛りは座標計算でassert検算。軸ラベルと目盛りは学習上必要な最小限。変域は網かけ＋端点●○。曲線は十分な分割数の折れ線近似で滑らかに描く。
2. **統計図系**: 五数要約・度数・割合は生データからプログラムで再計算してassert。架空データの図は「架空の練習用データ」の明示をキャプションまたは図内に置く。
3. **言語・カード図系**（英語・国語）: 文構造の箱図・語順パズル・カード型・吹き出しを枠＋ラベルで構成。人物はイラスト化しない。漢字の部品分解図はテキスト要素の組み合わせで作る（パスによる字形の自作をしない）。
4. **概念図系**(社会・理科ほか): 矢印フロー・対比パネル・関係図。社会は図でも政治的中立を守り、立場の優劣を視覚的に示唆する配置・強調をしない。理科の比喩に基づく図には「イメージ図」ラベルを付し、実測風に見せない。
5. **図を置く判断基準**: 文章説明が3文以上続く概念・手順・対比は図表化を検討する。「目が滑る」箇所の分節が目的——装飾目的の図は作らない。

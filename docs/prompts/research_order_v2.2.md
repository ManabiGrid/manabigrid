# 調査発注プロンプト（工程1用）v2.2

単元制作の最初の工程「調査」を、使用するAIに発注するためのテンプレートです。`{{変数}}` を差し替えて、**新しい会話（新規チャット）で**そのまま貼り付けて使います。工程全体の位置づけは [../OPS.md](../OPS.md) を参照してください。

このプロンプトは「調べる」だけを依頼します。**教材本文の執筆はまだ依頼しません**（調べる者と書く者を分けるのが本プロトコルの核です）。

```text
変数:
{{UNIT_TITLE}}   = 単元名（例: 平方根）
{{STAGE_GRADE}}  = 校種・学年・科目（例: 中学校第3学年 数学）
{{STRAND}}       = 領域（例: A 数と式）
{{PREREQ}}       = 前提単元
{{NEXT}}         = 後続単元
```

---

## 発注プロンプト（コピーして使う）

```text
あなたは学習教材プロジェクトの調査担当です。{{STAGE_GRADE}}「{{UNIT_TITLE}}」
（領域: {{STRAND}}）の執筆前調査を行い、調査ノート（research_notes.md）を
作ってください。教材本文はまだ書かないでください。

【鉄則】
1. 一次資料主義: 学習指導要領（平成29/30年告示）本文・同解説の該当箇所を
   実際に取得して読むこと。数値・文言はすべて「資料名＋年度＋URL＋確認日」
   付きの転記値として記録する。記憶からの数値記載は禁止。
2. 3分離: すべての記述を【確認済み】【推論】【未確認】のどれかに分けて書く。
   - 【確認済み】= 原典を実際に取得して該当箇所を読んだもの
   - 【推論】= 確認済み事実から論理的に導いたもの（根拠を添える）
   - 【未確認】= 取得できなかった・裏が取れなかったもの
   原典にたどり着けず他資料の孫引きしか確認できなかったものは
   【確認済み】にせず「引用の引用」と明記する。書誌情報（タイトル・
   発行年）だけ確認できたものを【確認済み】にしない。
3. 転載禁止: 市販教科書・過去問・調査問題の本文/図/数値設定を写さない
   （出題傾向・誤答の類型を自分の言葉でまとめるのは可）。
4. 課程コード照合: JP-COS LOD（jp-cos.github.io・CC BY 4.0）で本単元の
   学習指導要領コード候補を確認し、記録する（CC BY 4.0の出典表示を
   忘れない）。

【調べる項目】
1. 学習指導要領・解説での本単元の位置づけ。指定されている用語・記号の
   正確なリスト（教科書の慣用語との区別も）
2. 指導上の留意点・活動例・ICTへの言及
3. 公的調査（全国学力・学習状況調査など）での誤答傾向
   ——正答率・反応率は年度・設問番号つきの転記値で記録する。
   正答率と反応率は別の指標なので取り違えない。取れなければ「未確認」
4. 誤概念・つまずきの研究知見（公的資料＋可能なら学術論文2〜3本。
   アブストラクトまでしか読めなかったものはその旨明記）
5. 前提単元（{{PREREQ}}）・後続単元（{{NEXT}}）との接続
6. 一般的な配当時数・単元配列（公開されている年間指導計画の範囲で）

【出力の形】
- 冒頭に調査日・使用したAI名
- 本文は3分離（【確認済み】【推論】【未確認】）
- 末尾に source_list 表:
  資料名／URL／確認日／確認の深さ（本文まで・抄録のみ・書誌のみ）
- たどり着けなかった資料の表（試したURLを含む）。たどり着けなかったこと
  自体は失敗ではないので、隠さず記録して続行する
- 「調査しても残った未知」を5点まで正直に列挙
```

## 品質チェック（発注者側が受け取り時に見る点）

- [ ] source_list 表があり、URL・確認日・確認の深さが埋まっている
- [ ] 3分離ができている（全記述がどれかに属する）
- [ ] 「引用の引用」が【確認済み】に混ざっていない
- [ ] 統計値がすべて年度・設問番号つきの転記値になっている
- [ ] たどり着けなかった資料の表がある（空でも表自体はある）
- [ ] 教材本文を書き始めていない

主要資料が全滅（学習指導要領本文にすら到達できない等）した場合のみ、人間に相談してください。それ以外は記録して続行します。

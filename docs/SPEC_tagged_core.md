# 仕様: タグ付きcore原稿と5ビュー

本書は、本リポジトリの教材原稿の構造（タグ付きcore）と、そこから生成される複数ビューの仕様を定めるものです。

## 1. 原則

1. **単一の正（core）**: 本文は1回だけ書く。生徒用と講師用を別々に執筆しない（版どうしの齟齬リスクの根治）。
2. **講師層は「深い層」であって「裏方メモ」ではない**: 既存教科書の教師用指導書（生徒用テキスト＋加筆）と同じ段階構造。**自習者・意欲ある生徒が講師版を読むことを前提に書く**——生徒が読んで傷つく・興ざめする表現（「この誤答をする生徒は理解が浅い」等）を書かない。誤答の扱いは「よくある考え方とその修正」の語りで書く。
3. **本当に内部のものは機械可読層（contract）へ**: 来歴・検証台帳・内部判断は、学習者が読む層に置かない（講師ビューにも置かない）。
4. **ビューは生成物**: 手編集禁止。coreを直して再生成する。

## 2. core/ の構造

```text
core/
  unit.yaml        … unit_id・準拠課程コード（照合状態付き）・扱う範囲・扱わない範囲・レーン構成
  lessons/L01.md…  … タグ付き本文（下記タグ語彙）
  assets/          … 図版（SVG。生成スクリプト由来のみ——SPEC_figures.md参照）
```

### タグ語彙（ブロック単位・`:::` 構文。段階＝深さの順）

| タグ | 意味 | 生徒版 | 講師版 |
|---|---|---|---|
| （無印） | 本文コア（説明・例題・練習） | 掲載 | 掲載 |
| `:::zatsudan` | 先生の雑談枠（興味づけコラム） | 掲載 | 掲載 |
| `:::stretch` | 発展（本線外・指導要領内の限定的発展） | 設定により | 掲載 |
| `:::plus` | 私立・受験レーン差分 | レーン別ビューで | 掲載 |
| `:::guide` | 指導の深掘り層（なぜこの順で教えるか・よくある考え方とその修正・声かけ例）。**自習者可読の品質で書く** | — | 掲載 |
| `:::answer` | 解答・解説 | 演習用=—／自習用=独立ファイルへのリンク | 掲載 |
| `:::internal` | 内部メモ（原則使わない。使いたくなったら機械可読層へ移す合図） | — | — |

執筆上の注意:

- `:::guide` は「先生へのメモ」ではなく「一段深い解説」の文体で書く（自習者が読む前提）。
- タグの過剰分割をしない（1レッスンあたりguideブロック2〜4個が目安）。

## 3. views/（生成物・手編集禁止）

coreから生成器で5ビューに展開する。

| ビュー | 構成 |
|---|---|
| 生徒・印刷版 | 無印＋zatsudan（＋設定でstretch）。書き込み余白つき。解答の本文は含まない（対応解答ファイルへのリンクは全ビュー共通で末尾に付く） |
| 生徒・自習版 | 上記＋guide＋answer相当の導線（自習者は講師層まで読む、という設計判断を反映） |
| 講師版 | internalを除く全タグ＝生徒版の完全上位互換＋guide＋answer |
| ノート型AIツール投入版（演習用／自習用の2種） | 自己完結（位置参照なし・出典/ライセンス/免責を同梱・内部メタデータ除去） |

補足規約:

- **媒体等価**: 印刷版と画面版は同格の一次成果物。本文に媒体依存の表現を書かない。インタラクティブ教材には紙上の代替手順を併記する。
- **解答分離**: 自習用ビューでも解答は独立ファイル。各レッスン末に対応する解答ファイルへの明示リンクを生成器が挿入する。
- **準拠課程表示**: 全ビューの冒頭または末尾に、準拠課程（平成29/30年告示学習指導要領）と生成時点を自動表示する。
- レーン別ビュー: 公立標準レーンを基準に、私立・受験レーン差分（`:::plus`）を重ねて生成する。

## 4. 機械可読層（contract/）

散文の複製を持たない機械可読データの層。単元マニフェスト（unit_id・版・生成条件）、要確認事項（verify_required）台帳、誤概念の一覧（拡張コンテンツ生成の入口）、拡張ポイント定義、来歴情報を置く。

## 5. view_generator の役割と本文一致検査

- ビュー生成は `tools/view_generator/view_generator.py`（Python・標準ライブラリのみ・テスト付き）が行う。タグを解釈し、ビューごとのフィルタ規則で本文を取捨し、解答リンク・準拠課程表示などの定型要素を機械挿入する。
- **原稿（core）が正、ビューは生成物**という関係を守るため、全レッスン×全ビューを一括生成し、生成結果を検証する（generate → verify の全数実行）。
- 本文一致検査の考え方: 生成・差し込みなどの機械的な処理が教育的内容を変えていないことを、処理前後の正規化差分（体裁差を除いた本文比較）で確認する。ビューを手で直したくなったら、それはcoreを直して再生成すべき合図である。

## 6. 指導要領外コンテンツのラベル規約

- 指導要領外の拡張コンテンツは、unit.yaml に `outside_course_of_study: true` を必須とし、全ビューの冒頭に「指導要領外の拡張コンテンツ」ラベルを機械挿入する。
- `:::stretch`（単元内の限定的発展＝指導要領内）とは明確に区別する。
- 本リポジトリの核・骨格は指導要領ベースの単元群であり、拡張コンテンツは核に依存する周辺として配置する（再利用者が核と拡張を混同しない構造）。

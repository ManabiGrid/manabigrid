# 一次関数 教師ガイド（候補ドラフト）

> 【指導者向け】候補ドラフト（人間レビュー前）・自己学習用に配布可。解答の全文は `answer_key_supplement.md` に分離。

## 単元全体の設計意図

この単元は「ペース（a）」と「はじめの量（b）」の分離を一貫した軸にしています。L1で式の意味、L2で増え方の数値化、L3/L4で式とグラフの往復、L5で式の決定、L6で方程式との接続、L7で場面への適用と変域、という積み上げです。 〔根拠区分: Experimental〕

全レッスン共通の授業運用:

- 「例」を先に扱い、「たしかめ」は例の後。ヒントは即答させない。 〔根拠区分: Experimental〕
- 各レッスンの主目標は1つ。用語は用語ミニ整理の6語以外を増やさない。 〔根拠区分: Experimental〕
- prototypeは**教師の投影・確認用**が既定。生徒個別利用は未検証（ブラウザ・アクセシビリティテスト未完了）。

## L1 比例から一次関数へ

- **ねらい**: `y=ax` に初期量を加えて `y=ax+b` を作る。ペースとスタートの分離。
- **前提**: 中1の比例 `y=ax`。
- **板書の流れ**: ①空の水そう `y=3x` → ②「はじめ2L」を加えた表 → ③`y=3x+2` → ④2つの式の表を並べ「増え方は同じ、スタートがちがう」を可視化。
- **発問例**: 「1分後、2分後の量はどう計算した？」「2つの水そうで、変わらないのはどっち（ペース／スタート）？」
- **予想されるつまずき**: `+2` を「毎分2増える」と読む（生徒本文の「よくあるまちがい」に対応）。表で x=0 の行を指させて修正する。
- **先生用メモ**: 比例を `b=0` の特別な場合と位置づける一文は、L1で深追いせず言い切りで良い。
- **解答参照**: answer_key_supplement L1 ①②。

## L2 変化の割合

- **ねらい**: 値と増加量の区別。変化の割合 = Δy÷Δx。
- **前提**: L1の表の読み。
- **板書の流れ**: ①`y=3x+2` の表 → ②隣り合うyの差を書き込む → ③「割合」という言葉を最後に導入 → ④x増加量が1でない例（x: 2→5）。
- **発問例**: 「今見ているのは y の値？ それとも増えた量？」「`8÷2=4` はなぜ変化の割合ではない？」
- **予想されるつまずき**: `y÷x` 誤答。これはこの単元の代表的な定番誤答なので、誤答例を板書して否定する時間を取る価値がある。 〔根拠区分: Derived〕
- **先生用メモ**: 比例（b=0）に限れば x≠0 のとき `y÷x=a` が成り立つ。中1で使えた計算が一般の一次関数で使えなくなる境界として、生徒本文の「よくあるまちがい」末尾に補足がある。
- **prototype利用**: L2 rate-of-change table explorer（未公開・今後追加予定） — 生徒が手計算で1つ表を作った**後**に投影。Δx・Δy・Δy/Δxの対応を段階表示。記号量が多いため、最初から見せない。
- **解答参照**: answer_key_supplement L2 ①②。

## L3 式からグラフへ

- **ねらい**: 式→表→点→直線。a・bのグラフ上の意味。
- **前提**: 座標平面での点の取り方。
- **板書の流れ**: ①`y=2x+1` の表 → ②点を打つ → ③直線で結ぶ → ④`(0,1)` が切片、右1上2が傾き。
- **発問例**: 「最初に打つ点はどこが楽？（(0,b)）」「xを1増やしたら、点はどう動く？」
- **予想されるつまずき**: 切片をx軸上と誤読。y軸を指で押さえてから点を打たせる。
- **prototype利用**: L3/L4 graph slider（未公開・今後追加予定） — L3では**式→グラフ方向のみ**使う。読み取り機能はL4まで見せない（L3/L4混在は導線が重くなる。prototype suite reviewの指摘事項）。 〔根拠区分: Experimental〕
- **先生用メモ**: 用語「切片」は教科書で広く使われる慣用語だが、学習指導要領のコード表の指定用語ではない（コード表の〔用語・記号〕は「変化の割合」「傾き」の2語のみ）。指導では通常どおり使ってよい。
- **解答参照**: answer_key_supplement L3 ①②。

## L4 グラフから式へ

- **ねらい**: グラフ→切片・傾き→式。負・分数の傾きを含む。
- **前提**: L3。
- **板書の流れ**: ①切片2・右1上3のグラフ → ②式復元 → ③右下がりの例で負の傾き → ④格子点2つで分数傾き `2/3`。
- **発問例**: 「右へ進むと、上がる？下がる？」「読みやすい点はどれ？」
- **予想されるつまずき**: 負の傾きの符号落ち。「右へ→上がる/下がる→正/負」の言語化を全員に一度言わせる。
- **prototype利用**: 同じgraph sliderを**読み取りモード**で。L3と同時に両モードを見せない。
- **先生用メモ**: 分数傾きは「右へ3、上へ2」の移動語で支える。約分の練習にしない。
- **解答参照**: answer_key_supplement L4 ①②。

## L5 条件から式を決める

- **ねらい**: 傾き+1点、または2点から式決定。x座標が異なる条件の確認を第一手にする。 〔根拠区分: Experimental〕
- **前提**: L2の変化の割合、L4の傾き。
- **板書の流れ**: ①「決めるものはaとbの2つ→手がかりも2つ（この単元では「傾きと1点」「x座標が異なる2点」）」 → ②例1（傾きと1点） → ③例2（2点。最初にx座標を確認） → ④どちらの点で b を求めても同じことを一度見せる。
- **発問例**: 「まず何を求める？（a）」「x座標が同じ2点だったらこの方法は使える？」
- **予想されるつまずき**: 代入する点の座標の取り違え。`7 = 2×3 + b` のように、代入の形を毎回そのまま書かせる。
- **prototype利用**: なし（L5 prototypeは未作成。static優先。将来specは内部処理用ビュー（非同梱）に保管）。
- **先生用メモ**: 垂直な直線（x=k）はこの単元のcoreに入れない。質問が出たら「x座標が同じでyがちがう2点を通る `y=ax+b` は存在しないので、この決定問題の対象外」と返し、深入りしない（「別の方法なら求められる」という誤解を残さない）。 〔根拠区分: Rejected〕
- **解答参照**: answer_key_supplement L5 ①②。

## L6 二元一次方程式とグラフ

- **ねらい**: yについて解ける二元一次方程式をグラフ化し、交点=連立方程式の解と結びつける。
- **前提（重要・順序注意）**: 二元一次方程式／連立方程式の既習状況は学校の進度により異なる。**授業前に既習を確認すること。** 未習が多い場合は、「2つの式を同時に満たすxとyを探す」という言い方に留め、連立方程式の解法手続きには踏み込まない短縮運用が可能。
- **板書の流れ**: ①`2x+y=6` をyについて解く → ②`x+2y=8` も解いて2直線をかく → ③交点に印 → ④交点の座標を**両方の式に代入して**確認。
- **発問例**: 「交点はどんな点？（両方の直線に乗っている点）」「『x=2』だけで交点を答えたことになる？」
- **予想されるつまずき**: 交点をx座標だけで答える。移項時の符号ミス（`2y=-x+8` の段階を飛ばさせない）。
- **スコープ注意**: `x=k` は生徒coreに入れない。`y=k` も core の一次関数例として扱わない（teacher-side stretchの扱いは既知課題として人間レビュー待ち）。一般の連立方程式の解法網羅に広げない。 〔根拠区分: Rejected〕
- **prototype利用**: L6 intersection visualizer（未公開・今後追加予定） — **教師投影用**。交点→座標の組→代入確認、の順で画面を使う。直線を3本以上出さない。
- **解答参照**: answer_key_supplement L6 ①②。

## L7 一次関数の利用

- **ねらい**: 場面のモデル化と変域。「計算できる」と「場面で使える」の区別。
- **前提**: L1〜L5。（L6が短縮運用でもL7は実施可能。追いつき問題の「2つのyが等しい」はL6の考え方の再利用だが、差の縮まり方でも解ける。）
- **固定条件**: 水そうは 深さ40cm・初期4cm・毎分3cm・`y=3x+4`・`0 ≤ x ≤ 12` で固定。数値を変えない。
- **板書の流れ**: ①場面→式 → ②満水の時刻を方程式で → ③変域を不等式**と自然文の両方**で書く（「0分後から12分後まで」） → ④`x=15` は計算できるが場面で使えない例。 〔根拠区分: Experimental〕
- **発問例**: 「`x=100` を入れたら y はいくつ？ その水の深さは実際にあり得る？」
- **予想されるつまずき**: 変域を記号の暗記として扱う。必ず場面の言葉に戻す。ここでの変域は「場面で意味のある範囲」であり、形式的な定義域の理論には踏み込まない。
- **prototype利用**: L7 modeling/domain simulator（未公開・今後追加予定） — 変域外の値を入れたときの表示を見せる用途に限定。
- **解答参照**: answer_key_supplement L7 ①②。

## prototype利用の全体方針

- 4件（L2、L3/L4、L6、L7）とも未公開（レビュー中）。ブラウザ・アクセシビリティ・投影テストは**未完了**であり、授業実機投入の可否は人間レビュー後に判断。
- 既定は教師投影。生徒の個別操作は本パッケージでは想定しない。
- すべて印刷・静的fallback（本文中の表と、言葉によるグラフの説明）で代替可能。描画済みのグラフ図版は本パッケージには同梱していない。interactive無しでも単元は完結する。 〔根拠区分: Experimental〕

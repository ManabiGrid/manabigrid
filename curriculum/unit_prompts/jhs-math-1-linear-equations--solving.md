## 工程1: 調査（執筆はまだしない）

```text
あなたは学習教材プロジェクトの調査担当です。中学校第1学年 数学「一次方程式の解法」の
執筆前調査を行い、調査ノート（research_notes.md）を作ってください。
教材本文はまだ書かないでください。

【鉄則】
1. 一次資料主義: 学習指導要領（平成29/30年告示）本文・同解説の該当箇所を
   実際に取得して読む。数値・文言はすべて「資料名＋年度＋URL＋確認日」付きの
   転記値として記録する。記憶からの数値記載は禁止。
2. 3分離: すべての記述を【確認済み】【推論】【未確認】に分けて書く。
   原典にたどり着けず他資料の孫引きしか確認できなかったものは
   【確認済み】にせず「引用の引用」と明記する。
3. 転載禁止: 市販教科書・過去問・調査問題の本文/図/数値設定を写さない
   （出題傾向・誤答の類型を自分の言葉でまとめるのは可）。
4. 課程コード照合: JP-COS LOD（jp-cos.github.io・CC BY 4.0）で本単元の
   学習指導要領コード候補を確認し、記録する（出典表示を忘れない）。

【調べる項目】
1. 学習指導要領・解説での位置づけ、指定されている用語・記号の正確なリスト
   （教科書の慣用語との区別も）
2. 指導上の留意点・数学的活動等の例・ICTへの言及
3. 公的調査（全国学力・学習状況調査など）での誤答傾向
   ——正答率・反応率は年度・設問番号つきの転記値で。取れなければ「未確認」
4. 誤概念・つまずきの研究知見（公的資料＋可能なら学術論文2〜3本。
   アブストラクトまでしか読めなかったものはその旨明記）
5. 前提単元（方程式と等式の性質・一次式の計算）・
   後続単元（一次方程式の文章題）との接続
6. 一般的な配当時数・単元配列（公開されている年間指導計画の範囲で）

【出力の形】
- 冒頭に調査日・使用したAI名
- 本文は3分離
- 末尾に source_list 表（資料名／URL／確認日／確認の深さ=本文まで・抄録のみ・書誌のみ）
- たどり着けなかった資料の表（試したURL含む）
- 「調査しても残った未知」を5点まで正直に列挙
```

## 工程2: 正規化（調査ノート → 執筆用サマリー）

この工程が**いちばん事故が起きやすい**ところ。要約の途中で意味が変わると、後の工程では誰も気づけない。

```text
あなたは調査ノートの正規化担当です。添付の research_notes.md だけを材料に、
執筆者に渡す writing_summary.md を作ってください。
Webでの再調査はしない。ノートにない事実を足さない。

【意味を変えてはいけない6項目】（1つでも破ったらやり直し）
1. 「準正答」を「正答」に変えない
2. 「〜という誤答傾向がある」を「生徒は〜できない」という断定に変えない
3. 反応率・正答率の数値を「多くの生徒が」等のあいまいな頻度表現に変えない
4. 年度・設問番号・適用条件を落とさない
5. 例示を「代表例」「すべての場合」扱いにしない
6. ただし書き・除外条件・否定形（〜ではない）を落とさない

【必須の出力スロット】
- 単元の到達目標と指定用語（出典つき）
- つまずき・誤概念のリスト（出典・数値は転記値のまま）
- 執筆上の注意（ただし書き・条件をここまで生かす）
- 誤読禁止メモ: 「XをYと解釈しないこと」の形で列挙
  （例: 「反応率42%を『半数近くができない』と解釈しない」）
- 未確認事項のリスト（verify_required として執筆側に引き継ぐ）
```

## 工程3: 執筆

```text
あなたは独習用教材の執筆者です。添付の writing_summary.md だけを根拠に、
中学校第1学年 数学「一次方程式の解法」のレッスン群を書いてください。
サマリーにない事実の断定はしないでください（書きたければ verify_required の
注記つきで）。

【成果物】
1. lesson_map.md — 単元全体の設計（レッスン一覧・時間配当・前提と接続）
2. lesson_01.md 〜 lesson_NN.md — 本文（1レッスン=授業1〜2時間分）
3. answer_key_*.md — 解答は本文と別ファイルに分離。各レッスン末に
   対応する解答ファイル名を明記
4. 出典リスト（サマリーから引き継いだもの＋verify_required一覧）

【教材の設計ルール】
- 読者は「ひとりで学び直す中高生」。中間層〜苦手層を標準に、
  発展的な内容は stretch として区別する
- 学習者単独で完結: ペア活動・発表・インタビュー・家族の協力を前提にした
  活動を書かない。AIや検索の利用を促すのは可（その場合は検索フレーズ例・
  AI宛プロンプト例を併記し、丸投げ指示にしない）
- 画面と紙で等価: 「クリック」「タップ」等の媒体依存表現を書かない
- 各レッスンの冒頭に前提の確認（準備運動）、末尾に練習を置く
- 図表を積極的に使う（目安: 各レッスン1枚以上）。図は必ず
  「生成スクリプト→SVG」方式で作り、SVGの<title><desc>に図の意図と
  主要数値を書き込む（後から誰でも検証・再生成できるように）
- 数値例・データは自作する（実在の調査の数値設定を写さない）。
  作った数値は「教材用に作成した架空の値」と明示する

【文体】
- 親しみやすく、しかし正確に。呼びかけは「〜してみよう」を基本形とする
  （「〜してみて！」形は使わない）
- 「——」（ダッシュ）はここぞという時だけ（1レッスン1〜2回まで）。
  「そう、」「実は」の乱発・比喩の連打・体言止めの連続を避ける
- 重要な定義は【ことば】枠で明示。難しい漢字・固有名詞の初出には
  ふりがなを括弧書きで付ける

【安全規律】（違反した成果物は受け入れられません）
- 実在の人物（生徒・家族・特定個人）の個人情報を扱う活動を設計しない。
  例示人物は架空のみ。AI・検索の利用例でも実在個人の情報を入力させない
- 実在の逸話・歴史上の出来事を使う場合は出典を付け、事実確認できない
  ものは使わない。健康・災害・政治的評価に踏み込む雑談は書かない
- 市販教科書・過去問の本文/図/数値設定を転載しない
```

## 工程4: セルフQA（執筆とは別の会話で）

```text
あなたは教材の検査担当です。書き直しはせず、指摘だけを返してください。
添付: 教材ドラフト一式＋writing_summary.md。
各指摘は「箇所・違反したルール・修正案」の3点セットで。

【点検項目】
① 数値の年度なし使用がないか（すべての統計値に年度・出典があるか）
② サマリーにない事実断定がないか
③ 本文への解答漏れがないか（解答は別ファイルのはず）
④ 対象読者レベル外の内容（入試技巧・先取り）が本編に混ざっていないか
⑤ 扱わないと決めた範囲に踏み込んでいないか
⑥ 数値解答の再計算 — 全問を実際に解き直して一致を確認（最重要）
⑦ 他者前提の活動がないか（独習で完結するか）
⑧ 数値の指標名照合（正答率と反応率の取り違え等）＋時間配当の合計再計算
⑨ 文体点検（「してみて！」形ゼロ・——の頻度・初出ふりがな）
⑩ 中心例の意味検算 — 単元の中心例が主張する結論を、数値を変えた反例で
   検算する（値の一致だけでなく「主張の意味」が正しいか）
⑪ 量化・定義域・包含の検査 — 「どの〜でも」「必ず」「1つ分かれば決まる」型の
   主張に、0・境界値・退化ケースの反例チェックを行う

指摘がなければ「なし」と明言してください。
```

## 工程5: 提出

- QAの指摘を**執筆の会話に戻して**修正（検査担当に直させない）。
- 直ったら、CONTRIBUTING.md のチェックリストに沿ってPRへ。**使ったAIの名前と、どの工程を実行したか**をPR説明に書く。
- 最終検証（別系統AIの批判レビュー・人間レビュー）はメンテナ側で実施する。verify_required が残っていても提出して大丈夫——**残っていることが正直さの証明**。
